Unity: Integrating Firebase Realtime Database!

Posted on 2021-04-17 In Unity , Game Development

Firebase offers a complete SDK in order to easily integrate its different services (database, authentication, etc). We would be looking into integrating the realtime database in a simple unity project.

Pre-requisites

  • [✓] Unity 5.3 or later
  • [✓] VSCode(With unity extensions) or visual studio code with c# support
  • [✓] Firebase SDK installed

End Goal

To Demonstrate reading and writing from the firebase real-time database(from now on referred to as RTDB), We’ll be creating an application that would display the counter value directly from the database, and increase its value by one on clicking a button.

Steps

  1. Configure the Firebase Project
    1. Create a new firebase project, head to Realtime Database section and create a database. Select test mode for security rules while creating it.
    2. In the rtdb node, create a node “counter”. This would be from where we will be reading data from the unity project.
    3. (Optional) Re-download the google-services.json file and replace it in your unity project to ensure that the rtdb link is configured properly in unity.
  2. Setup Unity Project
    1. Ensure that the firebase database package is present in the unity project.
    2. Create a simple UI.
    3. Create a DBManager, that will control all the db connections.

In short: we have a listener attached to the counter node, and a function which sets the value(on clicking the button), So whenever this function is called, the value is updated in firebase. and As the value is changed, the listener function is also called which updates the UI

  1. Run the Project

If everything’s done right, the project should work and the value should be updating in the project as well as the database ^ _ ^

Possible Issues:

  1. The build environment is not set correctly. Is should be set to android as firebase sdk currently does not support native environments(Mac/Windows/Linux)
  2. Enable Firebase Realtime Database in the firebase project
  3. Firebase rtdb url may not be present in the google-services.json file

What are Functions in unity?

Functions is one of the fundamental concepts of programming language. A function is a block of code, that is meant to do a particular task and then return a value. Now some functions can do work but return nothing. On the other hand, some functions can do no work but return a value. Both the cases are possible and we will see a function in action with an example!

What are Co-routines in unity?

So as we saw functions do a task, and then return a value. It cannot return back to the main code block before completing the task.

Now, what if I want to delay the execution of my code? If I try delaying it in the main update thread, that would make our game slow and unresponsive. So I need a function that does some work, then continues normal work and then comes back and starts doing the work again.

This is what co-routines do. They are a bit tricky to understand but you’ll learn more about them as you develop your own applications and games.

Why Electron?

Posted on 2020-11-11 Edited on 2021-03-23 In Development , Javascript , Electron

But why Desktop apps at all?

Before we dive into cross-platform desktop apps, one might ask:

Why do we still need desktop apps? Isn’t Web Apps and PWA the future?

Well for most of the cases, it’s true. Web Apps have become very efficient and responsive now and can be preferred over desktop apps

……But, there are few use cases when it is absolutely required to build a desktop app. Desktop apps have a better system integration, are easier to access, and have better offline support than web apps.

Why Electron over other alternatives?

Electron provides a runtime to build desktop applications with pure JavaScript. You can think of it as a sort of web view that runs your application natively. The main advantages of Electron js are:

  • Easy to Learn: Electron JS is based on Javascript, so it’s easy for anyone who knows JS to understand the code. As Electron is based on web apps, it’s easier for web developers to get onboarded with the framework!
  • Has pool of APIs: You need not scratch your head to find the right feature that would help your implementation. The electron has a range of APIs which can be integrated easily into desktop apps saving the your time.
  • Great community support: One of the pros of being a popular framework is that it has an amazing community and can help you getting started with electron quickly!

Any Cons?

  • Huge file size: As electron is based on chromium, which means that each app you make will have it’s own version of chromium. So even extremely small, hello world applications can have a size of 100-200 mb!
  • Code Protection: The files aren’t encrypted which means that anyone can get a working copy of the code.
