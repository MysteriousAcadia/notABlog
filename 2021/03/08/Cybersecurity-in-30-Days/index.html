Ethical Hacking in 30 Days!

Posted on 2021-03-08 Edited on 2021-04-25

Introduction

Ethical hacking has been one of the skills I want to master for a long time. But I always find a way to procastinate and not start learning. That’s why I’ve taken up a “30 day Challenge” to start my journey in Ethical Hacking!

During these 30 days, I’ll be devoting atleast an hour everyday and try learning something in the field. I’ll try documenting my journey and post regular updates on my progress! With the help of a friend and few online resources, I’ve selected 3 courses to learn from and few websites, blogs and twitter accounts to follow.

Courses I plan to complete:

  • Basics of Ethical Hacking by Heath
  • Web Security by MIT
  • Web Security from PortSwigger

Websites I follow

  • https://tryhackme.com/
  • Pentester.Land
  • Intigriti

Weekly Updates

  • Week 1

    Started the Practical Ethical Hacking course offered by TCM security.
    Things I learnt in the first week:

    • Introduction to hacking, cyber security, red team, blue team, different fields and different aspects
    • Stages of analysis, installed the most popular tools and websites used by experts to do ethical hacking.
    • Installed Kali linux and brushed up on basics of terminal and bash scripting.
    • Learnt and explored about Shell Binding vs reverse shell binding.

  • Week 2

    It was a continuation of the course that I started in the previous week. My main focus was on trying to learn the implementation of tools and when it would be required.

    • Learned the usage and the syntax of most commonly used tools by people like Nmap, masscan, Metasploit, and Nessus
    • Solved few easy challenges on Kioptrix and watched a few walkthroughs.

  • Week 3

    With the course almost complete, I started off with the Stanford CS253 Web Security course which is taken by Feross(available on youtube). This course is intended to give an overview of web security, understand the common attacks, best practices, and have a better understanding of web security issues. During this week I explored:

    • HTML and Javascript
    • HTTP, Cookies, Sessions and Session Attacks
    • Same Origin Policy, its exceptions and Cross-site scripting(XSS)
    • Best practices against XSS

  • Week 4

    Completed the course started on web security. Also explored more into the topics I learned last week and these as well:

    • Privacy and Fingerprinting
    • DDoS, Phishing, Code Injection
    • TLS
    • Authentication(JWT, OAuth)
    • Server side security, best coding practices
    • DNS and DNS rebinding attacks.

Apart from these courses, I’ve also started to solve challenges on tryhackme and hackthebox!

Summary

I was able to devote atleast an hour everyday (some days it went as high as 4 hours!). Overall taking up this challenge helped me to block time from my schedule and learn a skill which I always wanted to learn. I’m still a beginner and still no where near the pros but its a good start and I will learn more and become better at it over the time!
